    Click to Call

    LivePerson’s Click to Call engagement provides brands with the ability to present an engagement on their website that offers consumers the option to receive a phone call from the brand’s representatives.

    The Click to Call solution works in the following manner:

    1. LiveEngage monitors the brand’s website and targets visitors based on the campaign logic specified by the brand in the LiveEngage UI.
    2. The Click to Call engagement is displayed to the targeted visitors. The creative definition is provided by the brand and implemented by LivePerson Professional Services.
    3. When a visitor clicks on the engagement, for example a banner, a page owned and hosted by the brand is opened with all the information required to start the voice call. See External conversion trigger for more details.
    4. [Optional] The brand may validate the request using LivePerson provided API, to prevent phishing calls. See Request Validation API for more details.
    5. A voice call to the visitor is initiated and managed by the brand.
    6. [Optional] The brand may pull data in real-time from LivePerson about the visitor and session to be used during the call. See Visitor Info API for more details.
    7. [Optional] The brand may report in real-time back to LivePerson on the call status and/or information about the visitor or the conversation. See Visitor Events (Engagement Attributes) API and Conversation Status Event (Engagement Attribute) for more details.

    This document outlines LiveEngage provided APIs mentioned in sections 3, 4, 6 and 7 above.

    APIs

    LiveEngage will provide the following APIs: >• External conversion trigger >• LivePerson Domain API >• Request Validation API >• Visitor Events (Engagement Attributes) API >• Conversation Status Event (Engagement Attribute) >• Visitor Info API

    All APIs except for the External conversion trigger are optional.

    External conversion trigger

    When a visitor the brand’s website clicks on a Click to Call engagement, a page owned and hosted by the brand is opened with all the information required to start the call, such as the number to call, LivePerson API callbacks, and identifiers to be sent to the callbacks. The structure of the URL is defined below.

    Request

    Method URL
    GET https://{brand_page_url}?v={version id}&aid={account id}&vid={visitor id}&sid={session id}&cid={context id}&eid={engagement id}&caid={campaign id}[&custom params]

    Query Parameters

    Note: These parameters are passed by LivePerson to a page owned and hosted by the brand.

    Name Mandatory Description Type/Value
    v Yes API version, e.g. v=1. This parameter is only used as a key for callback LivePerson API. double
    aid Yes LivePerson account ID. This parameter is only used as a key for callback LivePerson API. string
    vid Yes LivePerson visitor ID. This parameter is only used as a key for callback LivePerson API. string
    sid Yes LivePerson session ID. This parameter is only used as a key for callback LivePerson API. string
    cid Yes Engagement context ID (engagement impression id). This parameter is only used as a key for callback LivePerson API. long
    eid Yes LivePerson engagement ID (configuration). This parameter is only used as a key for callback LivePerson API. long

    Request Example

    https://{brand_page_url}?v=1&aid=123&vid=123&sid=123&cid=123&eid=1234&caid=123&cbnum=33555666777&mngnum=33555464123

    LivePerson Domain API

    This is a read-only API that returns the base domain of LivePerson that should be used in the APIs outlined in this document.

    Request

    Method URL
    GET http://api.liveperson.net/api/account/{accountId}/service/smt/baseURI.json?version=1.0

    URL Parameters

    Name Mandatory Description Type/Value
    account Yes LivePerson account ID string

    Response

    JSON Example

    {
 service: "smt",
 account: "1234",
 baseURI: "exampleDomain.liveperson.net"
}

    Elements in the Response

    Name Description Type/Value
    service Set to “smt” string
    account LivePerson Account ID string
    baseURI LivePerson domain to be used in the APIs outlined in this document string

    Request Validation API

    This is a read-only API that confirms the validity of the call trigger. This is a security measure to ensure the brand’s page was opened by LivePerson and not by phishing calls.

    Request

    Method URL
    GET https://{LivePerson domain}/api/account/{account}/monitoring/visitors/{visitor}/visits/current/campaigns/{campaign}/engagements/{engagement}/contexts/{context}?v=1&sid={sessionid}

    OAuth This API supports OAuth 1.0.

    Headers

    Parameter Description
    Authorization Contains token string to allow request authorization.

    Path Parameters

    Name Mandatory Description Type/Value
    v Yes API version string
    aid Yes LivePerson account ID string
    vid Yes LivePerson visitor ID long
    sid Yes LivePerson session ID long
    cid Yes Engagement context ID long

    Note: The above parameters are passed by LivePerson to a page owned and hosted by the brand. See External conversion trigger API section.

    Query Parameters

    Name Mandatory Description Type/Value
    v Yes API version double
    sid Yes LivePerson session ID string

    Request Example

    http status 200 OK

    Response Codes

    Code Description
    200 OK; Engagement is valid
    401 Unauthorized (no permissions)
    404 Engagement is not valid or resource not found
    422 Invalid Account ID
    500 Internal Server Error
    503 Service Unavailable

    Retry Policy Recommendation

    Name Description Recommendation
    4xx Client side error Do not retry, fix problem in code or appKey
    5xx Server side error Retry 3 times with a 5 second pause between retries

    Visitor Events (Engagement Attributes) API

    This API provides a way to report back to LivePerson consumer events such as the topic and resolution of the conversation during the session (e.g., call). This information is added to the existing session and can be used in the goal reached calculation.

    Request

    Method URL
    POST https://{LivePerson domain}/api/account/{account}/monitoring/visitors/{visitor}/visits/current/events

    OAuth

    This API supports OAuth 1.0.

    Headers

    Parameter Description
    Authorization Contains token string to allow request authorization
    Content-Type Set to “application/json” (encoded in UTF-8)
    x-http-method-override Set to “PUT”

    Path Parameters

    Name Mandatory Description Type/Value
    account Yes LivePerson account ID string
    visitor Yes LivePerson visitor ID string

    Note: The above parameters are passed by LivePerson to a page owned and hosted by the brand. See External conversion trigger.

    Query Parameters

    Name Mandatory Description Type/Value
    v Yes API version string
    sid Yes LivePerson session ID string

    Note: “sid” parameter is passed to by LivePerson to a page owned and hosted by the brand. See External conversion trigger.

    BODY/POST Parameters

    Parameter Description
    Engagement Attributes Array Array of Engagement Attribute. For more information about the available Engagement Attributes, refer to Engagement Attributes Overview document.

    Request URL Example

    https://lo.v.liveperson.net/api/account/1234/monitoring/visitors/xyz/visits/current/events?v=1&sid=rty.uio

    Request BODY Example

    [{
       "type": "lead", //MANDATORY
           "lead": {
           "topic": "inquiry about upgrade and iPhone 6s pricing", //TOPIC OR NAME OF A SUBMITTED LEAD
           "value": 200, //EVALUATED VALUE OF THE LEAD
           "leadId": "xyz123" //LEAD IDENTIFIER OR TICKET ID
       }
},
{
       "type": "service", //MANDATORY
       "service": {
          "topic": "add authorized user to the account ", // SERVICE ACTIVITY TOPIC OR NAME
          "status": 0, // STATUS ENUM
          // 0-Complete, 1-In Progress, 2-Approved, 3-cancelled, 4-Not Approved,
          // 5-Reviewed, 6-Missing Details, 7-Closed, 8-Removed, 9-Assigned, 
          // 10-Waiting for Customer Response, 11-Waiting for Response, 12-Pending, 13-Resolved 
          "category": "account maintenance", // SERVICE CATEGORY NAME
          "serviceId": "service12" // SERVICE UNIQUE IDENTIFIER OR TICKET ID
       }
}]

    Response

    Response Example

     http status 200 ok and empty json

    Response Codes

    Code Description
    200 OK; Operation performed successfully
    400 Bad Request; Problem with body or query parameters
    401 Unauthorized (no permissions)
    404 Not Found
    422 Invalid Account ID
    500 Internal Server Error

    Conversation Status Event (Engagement Attribute)

    This Engagement Attribute event provides a way to report back to LivePerson the status of the external conversation (e.g., voice call) for example, init, started, ended.

    JSON

     {
  "type": "extStatus",
  "extId": "externalId",
  "channel": "EXTERNAL",
  "status": "STARTED"/ "ACTIVE" / "ENDED"
}

    Visitor Info API

    This API provides a way to pull data in real-time about the visitor, engagement and session to be used by the agent during the call, for example, referrer, engagement attributes, etc. This information can be used to better route the call and provide more context to the agent.

    Request

    Method URL
    GET https://{LivePerson domain}/api/account/{account}/monitoring/visitors/{visitor}/visits/current/state

    OAuth

    This API supports OAuth 1.0 and OAuth 2.0 authentication patterns and requires SSL protocol.

    Query Parameters

    Name Mandatory Description Type/Value
    filter No Set to “agent” string
    v Yes API version string
    sid Yes Session ID string

    Note: “sid” parameter is passed by LivePerson to a page owned and hosted by the brand. See External conversion trigger.

    Request Example

    https://lo.v.liveperson.net/api/account/12345678/monitoring/visitors/222/visits/current/state?v=1&filter=agent&sid=123.567

    Response

    JSON Example

    See Appendix A: JSON Example.

    Elements in the Response

    Name Description Type/Value
    id Monitoring session ID string
    engagements Array of engagements displayed to the consumer array
    pages Array of the pages visited by the consumer array
    userAgent Consumer browser’s user agent string
    deviceOS Consumer’s operating system string
    deviceFamily Consumer’s device string
    lpv4 Consumer’s IP string
    geo Consumer’s Geo information (country, state, city etc.) object
    externalReferrer Referrer page string
    events Engagement Attributes events reported during the session array

    Response Codes

    Code Internal Code Description
    200 – OK; Operation performed successfully
    400 33 Bad Request; Problem with body or query parameters
    401 10 Unauthorized (no permissions)
    404 6 Invalid Account ID
    404 37 Invalid Visitor ID
    404 39 Invalid Session ID
    404 12 Not Found
    500 – Internal Server Error
